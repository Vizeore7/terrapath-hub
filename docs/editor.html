    Basics

    Set the title, author, language, and short catalog summary.

    What appears in the list

    The list view is meant to stay compact: title, class, popularity, language, and a short summary should explain the guide at a glance.

    • Use a clear build name, not a long paragraph.
    • Keep the summary focused on progression or playstyle.
    • Language should match the text inside the guide.

    Required mods

    Pick the content packs this guide depends on. Players can still see the guide without them, but the game can flag missing content.

    Class tags

    These tags drive filtering in the guide browser and in-game list.

    Guide tags

    Use broad, reusable tags like progression, bossing, pre-hardmode, or starter. Keep them general so search stays useful.

    Stages

    Split the guide into progression checkpoints instead of one long wall of text.

    Submission actions

    Review the rendered page first, then copy or download the JSON and open a GitHub issue.
